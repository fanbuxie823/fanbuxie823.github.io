5.3 变电站自动化系统信息模型

  • 信息交换机制主要依赖于定义明确的信息模型，这些信息模型和建模方法是IEC 61850系列的核心，IEC 61850系列使用该方法对实际设备中的通用信息进行建模，如下图

建模方法（概念性）

右侧的实际设备在图的中间建模为虚拟模型。逻辑设备中定义的逻辑节点对应于实际设备中的功能

  • 互操作的实现基于对定义的共同理解，因此描述数据模型的部分包含（认真考虑的）强制性语义表
  • 61850系列协议以独立于具体实现的方式定义了信息和信息交换，虚拟化提供了真实设备中与其他设备进行信息交换所感兴趣的那些方面的视图，本系列仅定义了提供设备互操作性所需的那些细节
  • 如Part 5所述，标准采用的方法是将应用功能分解成被用来交换信息的最小实体，称为逻辑节点（LN）
  • 多个逻辑节点可以构成一个逻辑设备，而单个逻辑设备通常在一个IED中实现，因为逻辑设备不是分布式的
  • 根据其功能，逻辑节点包含具有专用数据属性的数据列表（例如，位置）。他们具有明确的语义，服务通过特定的具体通信方式，根据IEC 61850-5中定义的规则和所要求的性能交换由数据表示的信息及其属性
  • 逻辑设备和逻辑节点的数据需要配置，为了从标准中选用合适的LN和LD，并赋予实例的特定值（比如实例之间的相互引用，交换机制，以及过程数据初始值）

5.5 数据的语义

由IEC 61850-7-4中定义的逻辑节点提供的特定数据的平均数量约为20，每个数据都包含几个详细信息（数据属性）

示例 XCBR

断路器开关的位置信息描绘成一棵树（概念上）

Pos
  • 逻辑节点XCBR中定义了断路器的位置（中间，关闭，打开或坏的状态，称为Pos）
  • Pos的定义为数据，分类为“控件”(Controls) ，可以通过使用控制服务(Control Services)和相关的服务参数(Service Parameters)来控制开关的位置（状态）
数据属性(Data Attributes)

  • Pos具有大约20个可通过不同服务访问的数据属性，就简单的RTU协议而言，他们可以分为3类：

    • 状态（或测量/计量值或设置）
    • 取代
    • 配置，描述和扩展

  • Pos具有的部分数据属性举例：

    • 实际断路器的位置（Pos.stVal）

    • 品质（Quality 状态值的当前有效性）

    • 时间戳信息（状态值的最后更改时间）

    • 发出命令的始发者（Originator）

    • 控制序号（Control Number）

    • 对控制响应为负值的原因诊断

stVal和品质的值可以远程替换，启用后立即生效

stVal，质量和时间戳（stVal相关联的）的当前值可以在IED的缓冲区中读取，报告或记录

服务参数(Service Parameters)

  • 服务参数(Service Parameters)不是数据模型的一部分：它们不代表数据属性，仅在命令执行的时候有效

5.7 服务映射到具体的通信协议

  • IEC 61850-7-2中定义的服务称为抽象服务。摘要意味着在IEC 61850-7-2中仅定义了描述服务请求的接收方和发送方所需动作的那些方面，他们基于IEC 61850-5中的功能要求。

  • 服务模型及其属性的语义以及在这些属性上运行的服务的语义（包括随服务请求和响应携带的参数）在IEC 61850-7-2中定义。

  • 特定通信服务映射（SCSM）中定义了特定的语法（格式），尤其是携带服务的服务参数的消息的编码以及这些参数如何通过网络传递，有两种映射方式：

    • 将服务映射到MMS（IEC 61850-8-1）以及TCP / IP和以太网（参见下图）之类的其他规定
    • 另一种是IEC 61850-9-2 –以太网上的直接映射。

5.8 自动化系统的配置

  • IEC 61850-6 定义了一种用来用于描述与通信有关的IED配置和IED参数，通信系统配置，交换站（功能）结构以及它们之间的关系的文件格式。称为SCL(Substation Configuration description Language )，基于XML语言

  • 此格式的主要目的是以兼容的方式在IED工程工具和不同制造商的系统工程工具之间交换IED功能描述和SA系统描述

SCL

  • SCL 可以描述：

    1. 就单线图而言的系统规范，以及将逻辑节点分配到单线的部分和设备，说明所需的功能
    2. 预配置的IED，含有：
      • 逻辑节点，数据集和报告控制块定义
      • 支持的服务：GOOSE，采样值，日志记录，文件处理
    3. 某种过程结构部分没有语义或预配置语义的预配置IED（例如双母线GIS线路馈线）
    4. 和个别过程功能以及一次设备绑定的所有IED的整个过程配置，进一步通过访问点连接和子网中可能访问的路径，来为所有可能的通信伙伴进行配置
    5. 与上项相同，但另外还可以在数据上逻辑节点之间的所有预定义关联和客户端服务器连接，如果IED无法动态建立关联或报告连接（在客户端或服务器端），则需要这样做

  • SCL 的目的：

    • SAS功能规范（上面1）
    • IED能力描述（上面2，3）
    • SA(Substation Automation)系统描述（上面4，5）

  • SCL 描述了

    • 在特定设备中使用了哪些可选信息
    • 所有逻辑节点的实例名称是什么
    • 存在哪些通信链接
    • IED与单线图的关系是什么
    • 所有需要用于系统工程的信息

  • 这些目的应以标准化的方式，支持系统设计，通信工程以及装置工程工具便捷地描述工程化系统通信

5.9 总结

由四个基本部件组成：

  1. 变电站自动化系统特定信息模型（Information Models）
  2. 信息交换方法（Information Exchange）
  3. 映射到具体通信协议（Mapping to MMS and TCP/IP/Ethernet）
  4. 变电站IED配置（Configuration File）

它们之间很大程度上彼此独立，意味着可以根据另一个应用程序域的要求，根据特定的灵活规则定义新的逻辑节点和新数据，轻松扩展信息模型，使用不同的通信技术栈，有关选择，请参见IEC 61850-8-x和IEC 61850-9-x

6 IEC 61850系列的建模方法

6.1 应用功能和信息的分解

  • 正如IEC 61850-5所描述的，该系列的通用方法是将应用功能分解为最小实体，即逻辑节点，用于通信，并将它们合理的分配到专用设备
  • 根据其功能，这些逻辑节点包含具有专用数据属性的数据
  • 数据类的定义方式：容易理解，都能接受

6.2 用逐步合成方法创建信息模型

  • 根据61850-5定义的要求，61850-7.4、61850-7.3和61850-7.2定义了如何建模变电站中的信息和通信

  • 建模采用逻辑节点主要地作为基本组成部件去合成变电站自动化系统的可视信息，用于描述由应用产生和使用的信息以及和其他IED进行信息交换

  • IEC 61850-7-4xx 系列标准对 IEC 61850-5中的逻辑节点和数据进行了更精确的定义，使用面向对象的方法定义了LN和他们的数据的内容（语义）和形式（语法），并用表的形式定义了逻辑节点和数据类(7.4)、公用数据类(7.3)和服务模型(7.2)

  • 逻辑节点集合几个数据类来构成特定功能

  • 比如XCBR逻辑节点包含超过15个数据类，代表了实际断路器的公共信息，并可重复用来去描述各种类型的断路器的公共信息，表2是XCBR的一个概念性的简短描述

  • 输出模型(Output Model)服务仅对内部过程影响，通过过程接口向过程输出信号，或改变数据属性状态值而触发报告，如果过程接口是一个IED，这个服务直接向过程发送输出信号
  • 输入模型(Input Model)定义了很多服务，输入信息直接从过程接口输入信息或在IED中计算
  • 在线管理(Online Management)，有些服务可以在某种程度上用于远程管理IED，例如定义数据集，将引用设置为特定值或通过报告控制块发送特定报告
  • 信息模型（逻辑节点和数据类）和服务模型提供了检索有关信息模型和对信息模型进行操作的服务（自我描述）的全面信息的方法

  • 分层信息模型中的很多数据属性可以用设定服务设定
  • 数据属性的设定值通常仅受应用程序约束
  • 对于拥有控制块属性的各种控制块(SGCB，BRCB，LCB)，控制块属性的设定值受相应控制块的状态机的约束
  • 控制块根据其属性集的值运行，还可以使用SCL文件或通过其他本地方式配置这些值，所有属性值可以被其他IED读取

设定数据

  • 设定数据（setting data, 例如AVCrv，TmACrv，TmMult等）具有与设定组(setting groups)定义的一样多的值
  • 设定组是一组拥有相同内容的设定集

6.4.3 输入模型

6.4.3.1 输入模拟信号采集

模拟值输入模型(第一步)

  • 一般来说，原始信号通过信号调节器调节
  • 在61850中，模拟信号在转换为数字信号之前不作为数据
  • 采样率（可配置数据的数据属性smpRate）确定应多久采样一次该值
  • 可以从通过通信链路传输的样本值中直接获取原始数字值
  • 处理信号的方法可以通过设置数据ClcMth来指定（真RMS，峰值幅度等），没有ClcMth的情况下，计算方法应视为UNSPECIFIED，表示UNKNOWN
  • 值用来通讯(建模为特定数据属性instMag)之前需要满足一些条件：
    • 数据的取代 / 非取代开关（数据属性subMag）
    • 操作员闭锁/非闭锁开关
  • 第一步的结果是带有相应质量信息的“中间值”（仍为模拟值）
6.4.3.2 数据属性值处理、监视和事件检出

模拟值输入模型(第二步)

  • 中间值有多种用途：
    1. 提供数据的瞬时属性值（例如上图的幅度）
    2. 计算死区值(mag)
    3. 确定当前域
  • 瞬时属性值(instMag)
    • 数据属性名为instMag，功能约束FC=MX(Measurands)，瞬时值没有触发选项
  • 死区值(mag)
    • 死区值应基于instMag的死区计算，如下图

      范围和死区值
    • 按instMag的配置参数值db计算后，如果值已改变，则以instMag当前值刷新mag
    • 死区配置db的值应表示最大和最小参数值之间的差异百分比，单位为0.001％（数据的粗细度？）
    • 注意db值与由模拟变送器的精度和由A / D转换的精度定义的数据的精度无关（转换后数据的二次采样？）
    • mag值更改时，都会创建一个内部事件。 死区值mag和事件（数据更改–根据触发选项TrgOp = dchg）可用于进一步的操作，例如报告或记录
  • 当前域
    • 通过监视中间值来确定值的当前域
    • 任何时候如果 instMag 的值超过值域，会产生内部事件，值域和时间可以被用于下一步行动（例如报告和记录）
    • 属性rangeC用于配置与不同限制关联的参数(min,max等)
    • 品质和t（时间戳）这两个属性随时可用

6.4.3.3 数据报告和记录
6.4.3.3.1 总体

报告和记录模型

  • 内部事件为报告和记录的触发基础
  • 信息分组构成数据集(data set)，数据集是报告和记录内容的基础
  • 数据集包含数据值和数据属性值的引用，并指定了那些数据和数据属性需要报告和记录

  • 日志控制块

缓存报告控制块

上图展示了基本缓存报告控制块的原理：

  • 缓存和非缓存报告是由报告控制块配置开始的，设置enable buffer attribute为TRUE就开始报告；反之停止报告

  • 如果有触发选项，例如通信中断，缓存报告控制块会继续储存事件数据。通信再次可用时，报告会继续。

  • 缓存报告控制块会在某些实际限制下(缓存大小，最大中断时间)，保证按事件顺序(SoE)传送

  • 非缓存报告控制块在通信中断时不支持SoE

  • 缓冲报告控制块具有几个控制报告过程的属性(RptID、RptEna、DatSet…)

  • 缓冲时间

    如上图所示，如果在第一个事件之后不久发生了其他几个事件，则服务器可以使用“缓冲时间”属性来减少报告的数量。在此期间发生的更改，将导致在缓冲时间结束时生成一个包含所有更改的报告

报告允许仅发送值，而无需数据和数据属性的任何对象引用，然后可以从数据集定义中检索对象引用(如下图)，还可以将数据和数据属性的对象引用与数据一起发送

数据集成员和 包含-位字符串

  • 如果(1)没有与值一起发送对象引用和(2)仅报告了数据集成员的子集的值，则应提供一种方法来确定报告的值属于哪个成员。
  • IEC 61850-8-1中定义的SCSM定义了一个包含-位字符串( inclusion-bitstring)，以指示数据集的成员。 数据集成员的顺序与数据集中定义的顺序相同
  • 包含-位字符串( inclusion-bitstring)的 0 和 1 表示该位置的属性值是否报告，上图中为1 0，表明该第一个成员属性报告，而第二个成员属性不报告
6.4.3.3.3 数据记录

记录模型提供了一个储存值(日志条目)的日志，日志控制块控制哪些数据值什么时候存入日志，而能储存的条目数目取决于日志条目的容量和储存器的容量

日志控制块概念图

上图是包含一个日志和三个日志控制块的例子：

  • 首先需要配置和启用日志控制块（和缓存报告控制块一样）
  • 启用后服务器可能会关闭
  • 日志条目到达时将被存储到日志中，日志按时间顺序存储， 这允许之后按事件序列（SoE）检索
  • 日志随时可以启用，不同的日志控制块允许将来自不同数据集的信息存储到一个日志中，每个日志控制块均独立于其他控制块
  • 日志控制块中同样有一些控制过程的属性(LogEna,DatSet…)
6.4.3.4 对等数据值发布

对等通信提供了：

  • 通用变电站事件交换服务（基于GOOSE的多播）
  • 采样测量值交换服务（基于多播或单播）

对等通信数据值发布模型

  • GOOSE模型同样具有多个控制发布过程的属性(GoEna,GoID…)

  • 逻辑节点LTRK包含IEC 61850-7-2中定义的任何服务所需的数据类，数据类基于同样在IEC 61850-7-2中定义的通用数据类别(CDC)

  2. 如何访问功能以及（一般而言）如何交换信息（请参见IEC 61850-7-2）
  3. 设备如何连接到通信网络（IEC 61850-8-x和IEC 61850-9-x系列）

IEC 61850-7-4包含超过2000个已命名和定义明确的信息元素的列表，这些信息元素使得能够创建真实的变电站设备的信息模型：该静态信息模型继承了IEC 61850-7-3的类型信息，并继承了IEC 61850-7-2所需的（动态）通信服务。

设备的动态行为

  • 真实设备的动态行为是通过实现的信息模型的可配置属性以及更改其可变值来建立的
  • 在标准中定义了信息模型中任何值更改所产生的影响。控制“ Circuitbreaker1”的结果是，实际电路开关断开或闭合
  • 控制器可以立即将报告（值，质量和时间戳）发送给启动器，并且可以将此事件另外写入设备的日志中以供以后检索
  • 控制器的各种动态行为可以由工程工具预先配置，通过使用用于远程配置（设置）的特定服务配置控制器可以更改行为。例如，可以远程启用或禁用报告值

7.2 建模第一步：逻辑节点和数据

  • IEC 61850-7-4 定义了很多逻辑节点，例如下图的断路器XCBR

逻辑节点和数据

Pos代表开关的位置，Loc表示”本地操作”，用来限制远程操作

  • 逻辑节点由很多代表应用特定意义的数据组成（详见附录A），数据构成了网络上大多数信息交换的基础，与设备的大多数交互是通过逻辑节点和服务中的数据进行的
  • 每个公共数据类都有为其分配的服务，这些服务定义了允许对这些数据进行操作的可能服务
  • 功能约束（FC）为特定数据类的每个信息定义了读写控制，数据信息定义为强制性或可选性
  • 逻辑节点名称和数据名称定义了变电站设备的标准化含义（语义）。信息模型包括许多逻辑节点，数据和数据属性
  • 该模型也是变电站配置语言SCL（详见IEC 61850-6 或 SCL）的基础 ，实例从其类继承所有内容，并为其分配一个唯一的名称
